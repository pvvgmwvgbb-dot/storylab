# Project Philosophy

StoryLab stores the path from an abstract idea to a visible learning experience.

The important asset is not the slide. The important asset is the teaching transformation.

A lesson should preserve:

- what is hard to understand
- what scene makes it visible
- what action lets the learner discover the pattern
- what question the teacher should ask
- what abstraction should remain at the end

## Image first, text second

StoryLab is designed for teacher-led learning. The screen should act as a visual anchor. The teacher gives the live explanation.

That means a good StoryLab scene should usually have:

- one strong image
- one clear situation
- very little text
- one question or action

## Output is replaceable

The same lesson source may later become HTML, PPT, PDF, video, or an interactive game.

The output format can change. The teaching path should survive.
