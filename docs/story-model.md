# Story Model

StoryLab uses a simple unit called a scene.

A scene is not a normal slide. A slide usually presents information. A scene creates a situation.

## Scene fields

Recommended fields:

- `id`: stable scene identifier
- `visual_anchor`: what should be visible on screen
- `purpose`: why the scene exists
- `teacher_prompt`: what the teacher may say aloud
- `discovery`: what the learner should notice

## Common lesson flow

```text
wonder -> situation -> problem -> visual model -> action -> discovery -> abstraction -> transfer
```

This is only a draft. The model should evolve after real lessons are tested with children.
